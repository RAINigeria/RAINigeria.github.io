What You Will Build

By the end of this session you will have set up the complete infrastructure below — a professional, AI-powered training management portal with automatic assignment marking. Every component is either free or low-cost.

◈

The Three Pillars of Low-Data AI

Before we build anything, you need to understand why this is possible with limited data. These three techniques are what make it work.

01
Transfer Learning
Borrow the brain that someone else already trained. Take a model trained on millions of examples and adapt it to your specific, smaller task. Like hiring an experienced engineer and teaching them your company's system — instead of training someone from childhood.
02
Prompt Engineering
Get maximum intelligence from existing AI by crafting precise instructions. A well-written prompt can replace months of model training. No code required. No dataset required. Just the right words, in the right structure.
03
Synthetic Data
When you don't have enough real data — generate it. Use AI to create realistic training examples. Especially powerful in Africa where large labelled datasets for our languages, crops, and diseases rarely exist.
💡
Key Insight: These three techniques together mean you can build functional, useful AI systems without a supercomputer, without millions of naira, and without a PhD. Everything in this workshop uses free or near-free tools. The only investment required is your time and your curiosity.

Module 1 — GitHub Setup
1
Create Your GitHub Account
~5 minutes

GitHub is the home of your code. It is also the platform that will host your public website for free. Think of it as Google Drive — but for code. Everything you build goes here first.

ℹ️
What is GitHub? GitHub is a website where developers store and share code. Over 100 million developers use it worldwide. It is owned by Microsoft. The basic account is completely free and gives you everything we need today.[1]
GitHub Sign-Up Page — github.com
  1. Open your browser and type github.com in the address bar. Press Enter.
  2. On the GitHub homepage, click the Sign up button in the top-right corner of the page.
  3. In the Email address field, type your email address (Gmail works perfectly).
  4. In the Password field, create a password of at least 8 characters. Write it down somewhere safe — you will need it again.
  5. In the Username field, choose your username carefully. This will become part of your website address. Use something professional like drAyoolaRAIN or your institute's name. No spaces allowed — use hyphens instead.
  6. Click the Continue → button.
  7. GitHub will show you a small puzzle to confirm you are human. Complete it (usually drag a piece or click a pattern).
  8. Click Create account.
  9. GitHub will send a verification code to your email. Open your email, find the code (6 digits), and type it into the box on screen.
  10. GitHub will ask "How many people will be using GitHub?" Select Just me.
  11. It may ask about your role. Select Teacher or Student — this can unlock free premium features for educators.[2]
  12. Select the Free plan. Click Continue for free.
  13. You are now inside GitHub. Welcome! 🎉
⚠️
Username Warning: Your username cannot easily be changed later and will appear in your website's URL. Avoid numbers, underscores, or anything unprofessional. Example of a good username: rain-institute or olusola-ayoola. This becomes: rain-institute.github.io
2
Create Your Special Repository
~5 minutes

A repository (repo) is like a folder for your project. We are going to create one with a very specific name that tells GitHub to automatically turn it into a public website.

🔴
Critical: The repository name must be exactly: YOURUSERNAME.github.io — replacing YOURUSERNAME with your actual GitHub username. If you type it wrongly, the website will NOT work. Double-check every character.
Creating a New Repository — github.com/new
  1. Once logged in, look for the + icon in the top-right corner of GitHub. Click it.
  2. A dropdown menu appears. Click New repository.
  3. Under Repository name, type exactly: YOURUSERNAME.github.io — where YOURUSERNAME is your actual GitHub username. So if your username is rain-institute, type rain-institute.github.io.
  4. Below the name field you should see a green message: "✓ [name] is available." This confirms your name is typed correctly.
  5. Under visibility, select Public. This is required for free GitHub Pages hosting.
  6. Scroll down and tick the checkbox next to Add a README file. This creates the repository with at least one file inside it (GitHub requires at least one file).
  7. Click the green Create repository button at the bottom.
  8. GitHub will take you to your new repository page. You will see a single file called README.md. Your repository is created! ✓
3
Enable GitHub Pages — Your Free Website
~3 minutes

GitHub Pages turns your repository into a live public website — automatically. Once enabled, any HTML file you add to your repository will be instantly visible at yourusername.github.io to anyone in the world.

GitHub Pages Settings — Settings → Pages
  1. Inside your repository, click the Settings tab. It is in the row of tabs near the top of the page (after Code, Issues, Pull requests...).
  2. On the Settings page, look at the left sidebar. Scroll down until you see Pages. Click it.
  3. Under Build and deployment → Source, click the dropdown and select Deploy from a branch.
  4. Under Branch, click the first dropdown and select main.
  5. The second dropdown should say / (root). Leave it as is.
  6. Click the Save button.
  7. The page will refresh. After about 1–2 minutes, a green box will appear at the top saying: "Your site is live at https://YOURUSERNAME.github.io"
  8. Click that link. Your website is now live on the internet! 🌍 (It will show a blank or basic page for now — we will add content in the next step.)
💡
Patience Required: After clicking Save, it can take up to 2 minutes for GitHub to deploy your site. If the green box doesn't appear immediately, wait 2 minutes and refresh the page. Do not click Save multiple times.
Module 2 — Building Your Website
4
Build Your Training Portal Page Using AI
~15 minutes

You do not need to write code from scratch. You will ask AI to write it for you, then upload it to GitHub. This is prompt engineering in action.

🤖
Which AI to use? Use any of the following: Claude (claude.ai), ChatGPT (chat.openai.com), or GitHub Copilot (built into GitHub). All three can write excellent HTML code. Claude and ChatGPT are free to use on their basic plans.

Copy and paste this prompt into Claude or ChatGPT:

📋 Your AI Prompt — Copy This Exactly
Write me a complete professional HTML page for a training institute called [REPLACE WITH YOUR INSTITUTE NAME]. The page should have: (1) a navigation bar with links to Home, Courses, About, and Contact; (2) a hero section with a bold headline and a "Enroll Now" button; (3) a section showing three course cards — one for AI Fundamentals, one for Data Science, and one for Robotics; (4) a footer with contact information. Use modern CSS inside the same file (no external files). Make it fully responsive for mobile phones. Use a professional blue and white colour scheme. Include hover effects on the buttons and course cards. Make it look like a real Nigerian educational institution website.
  1. Go to claude.ai or chat.openai.com in a new browser tab. Log in or create a free account.
  2. Copy the prompt above (after replacing [YOUR INSTITUTE NAME] with your actual institute name).
  3. Paste it into the AI chat and press Enter. Wait about 30 seconds.
  4. The AI will generate a complete HTML file. You will see a large block of code starting with <!DOCTYPE html>
  5. Click the Copy button (usually appears at the top of the code block). All the code is now in your clipboard.
  6. Go back to GitHub and open your repository.
  7. Click the Add file button, then click Create new file.
  8. In the filename box at the top, type exactly: index.html
  9. Click inside the large text area below and paste your code (Ctrl+V or Cmd+V).
  10. Scroll to the bottom and click Commit changes.
  11. A dialog box appears. Leave everything as is. Click Commit changes again.
  12. Wait 1–2 minutes, then visit https://YOURUSERNAME.github.io to see your live website!
💡
Not happy with the design? Simply go back to the AI and say: "Make the hero section bigger and use a green accent colour instead of blue." The AI will regenerate the code. Repeat until you are satisfied. This is iterative design — the same way professional developers work.
5
Attach a Custom Domain Name
~10 minutes

Your site works at yourusername.github.io — but for a professional institution, you want something like rain-institute.edu.ng or mytraining.ng. Here is how to get one and connect it.

Where to Buy Good For Price (Approx) Website
Whogohost .com.ng, .ng, .com domains ₦3,000 – ₦7,000/year whogohost.com
QServers .com.ng Nigerian domains ₦2,500 – ₦5,000/year qservers.net
GoDaddy International .com domains $10 – $15/year godaddy.com
  1. Go to whogohost.com (recommended for Nigerian institutions). Search for your desired domain name in the search box.
  2. Select your domain and complete the purchase. You will need to create an account and pay via card, bank transfer, or USSD.
  3. Once purchased, log into Whogohost. Find your domain in My Domains. Click Manage DNS.
  4. Add a new DNS record. Type: CNAME. Name: www. Value: YOURUSERNAME.github.io. Click Save.
  5. Add another record. Type: A. Name: @. Value: 185.199.108.153. Click Save. (This is GitHub's server address.)[3]
  6. Go back to GitHub → your repository → Settings → Pages.
  7. Under Custom domain, type your new domain name (e.g., www.rain-institute.ng). Click Save.
  8. GitHub will verify your domain. This can take 10–30 minutes. Once verified, a green tick will appear and your custom domain is live!
Module 3 — Backend Server
6
Create a Render Account — Your Free Server
~5 minutes

Your GitHub Pages site is just the front door — the beautiful public website. The backend server is what handles student logins, stores data, processes payments, and connects to AI. Render hosts your server for free.[4]

ℹ️
Why Render? Render is a cloud hosting platform that is free for basic use, very easy to set up, and automatically re-deploys your server every time you update your code on GitHub. No server management required. No monthly fees on the free tier.
  1. Go to render.com in your browser.
  2. Click the Get Started for Free button.
  3. Click Continue with GitHub. This is the easiest option — it links Render directly to your GitHub account so they can talk to each other.
  4. GitHub will ask you to confirm. Click Authorize Render.
  5. You are now inside the Render dashboard. It will look empty — that is fine. You will add your first service in the next step.
💡
Important — Free Tier Limitation: On Render's free tier, your server "sleeps" after 15 minutes of inactivity. The first request after sleep takes about 30 seconds to respond. For a training portal with regular use, this is acceptable. If you need it always-on, the paid tier starts at about $7/month.
7
Connect Render to GitHub — Auto-Deploy Your Server
~10 minutes

This is the magic step. Once connected, every time you push new code to GitHub, Render automatically updates your live server — no manual uploading required. Here is how to set it up.

⚠️
Before this step: You need to have your server code in a GitHub repository. In Step 10, we will use AI to write the server code. For now, create a second private repository on GitHub called training-portal-backend. This is separate from your website repository.
Render Dashboard — Creating a New Web Service
  1. In the Render dashboard, click the New + button in the top right.
  2. Select Web Service from the dropdown.
  3. Under "Connect a repository," you will see your GitHub repositories listed. Click Connect next to your training-portal-backend repository.
  4. Fill in the Name field: type something like training-portal-api.
  5. Set Region to Frankfurt (EU Central) — this is geographically closest to Nigeria and will give the fastest response times.
  6. Set Environment to Python 3.
  7. Set Build Command to: pip install -r requirements.txt
  8. Set Start Command to: python app.py
  9. Scroll to the bottom and select the Free plan.
  10. Click Create Web Service. Render will begin deploying. This takes 2–5 minutes the first time.
  11. Once deployed, you will see a green "Live" badge and a URL like training-portal-api.onrender.com. This is your server's address.
🔴
Setting Environment Variables (Secrets): Your server needs secret keys for the database, email, and payments. Never put these in your code. In Render, click Environment on the left sidebar of your service, then click Add Environment Variable for each secret. We will add these in the steps below.
8
Set Up Your Database with Neon
~8 minutes

Your database is where all your data lives — student accounts, course enrollments, assignment submissions, payment records, and grades. Neon provides free PostgreSQL database hosting that connects easily to your Render server.[5]

  1. Go to neon.tech in your browser. Click Sign Up Free.
  2. Sign up with your GitHub account (recommended — click Continue with GitHub).
  3. Neon will ask you to create a project. In the Project name field, type: training-portal. Click Create project.
  4. Neon will create your database and show you a Connection string. It looks like: postgresql://user:password@ep-xxxx.eu-central-1.aws.neon.tech/neondb
  5. Click the Copy button next to this connection string. Keep it safe — this is your database address.
  6. Go back to Render → your web service → Environment.
  7. Click Add Environment Variable. In the Key field type: DATABASE_URL. In the Value field, paste the connection string you copied from Neon.
  8. Click Save Changes. Your server now knows how to connect to your database.
💡
Neon Free Tier: The free tier includes 512MB storage and enough compute for a training institute with hundreds of students. You can store thousands of student records, thousands of assignments, and years of data within this limit. When you outgrow it, upgrading costs about $19/month.
Module 4 — Services & AI
9
Add Email with Mailtrap
~5 minutes

Your portal needs to send emails — welcome messages when students register, password reset links, assignment feedback notifications. Mailtrap provides a free email service for exactly this purpose.

  1. Go to mailtrap.io. Click Sign Up Free.
  2. After logging in, click Email Sending in the left sidebar, then click Sending Domains.
  3. Click Add Domain. Enter your domain name (e.g., rain-institute.ng). Follow the instructions to verify it — this involves adding DNS records in Whogohost (similar to what you did for GitHub Pages).
  4. Once verified, click SMTP/API Settings. You will see your SMTP credentials: Host, Port, Username, and Password.
  5. Go back to Render → your service → Environment. Add these environment variables:
    • MAIL_HOST = smtp.mailtrap.io
    • MAIL_PORT = 587
    • MAIL_USER = (your Mailtrap username)
    • MAIL_PASS = (your Mailtrap password)
  6. Click Save Changes. Your server can now send emails.
10
Build Your Server with AI Prompt Engineering
~20 minutes

Now we use AI to write the actual server code. You will prompt Claude or ChatGPT to write a complete Python Flask server. You do not need to understand every line — you need to understand what it does and be able to modify it.

📋 Server Code Prompt — Copy and Paste into Claude/ChatGPT
Write me a complete Python Flask web server for a training management system. Use PostgreSQL with psycopg2. The server must have these endpoints:

POST /api/register — accepts email, password, full_name. Creates a new student account. Sends a welcome email via SMTP. Returns success or error JSON.

POST /api/login — accepts email and password. Returns a JWT token if valid, or an error.

POST /api/forgot-password — accepts email. Generates a reset token, stores it, and sends a reset email.

GET /api/dashboard — requires JWT token in Authorization header. Returns student profile and their enrolled courses.

GET /api/courses — returns all available courses as JSON.

Use environment variables for: DATABASE_URL, JWT_SECRET, MAIL_HOST, MAIL_PORT, MAIL_USER, MAIL_PASS. Include full error handling. The server should run on port 5000. Include a requirements.txt file listing all dependencies.
  1. Copy the prompt above. Paste it into Claude or ChatGPT. Press Enter.
  2. The AI will generate two files: app.py (the server) and requirements.txt (the list of packages needed).
  3. In your training-portal-backend GitHub repository, click Add file → Create new file.
  4. Name it app.py. Paste the server code. Click Commit changes.
  5. Create another new file. Name it requirements.txt. Paste the requirements content. Click Commit changes.
  6. Go to Render. Your service will automatically detect the new code and begin redeploying. Watch the Logs tab — within 2-3 minutes you should see "Server running on port 5000".
🔴
Add JWT_SECRET to Render Environment: Go to Render → your service → Environment. Add: JWT_SECRET = any long random string of letters and numbers (e.g., "Rain2026SecureKey$xK92mP!"). This is the key that signs your login tokens. Keep it secret and never share it.
11
Add Nigerian Payments with Paystack
~10 minutes

Paystack is Nigeria's leading payment gateway. It accepts card payments, bank transfers, USSD (no internet needed on student's phone), and mobile money. You can start receiving payments with a personal account — no business registration required to get started.[6]

  1. Go to paystack.com. Click Create a Free Account.
  2. Fill in your details. Use your institute's email. Click Create account.
  3. Check your email for a verification link. Click it to verify your account.
  4. Once inside the Paystack dashboard, click Settings → API Keys & Webhooks.
  5. You will see two keys: Test Secret Key and Live Secret Key. For testing, use the Test key. When ready to go live, switch to the Live key.
  6. Copy your Test Secret Key (starts with sk_test_).
  7. In Render → Environment, add: PAYSTACK_SECRET_KEY = paste your key. Click Save Changes.
  8. Ask AI to add payment integration to your server: "Add a Paystack payment endpoint to my Flask server. POST /api/enroll — accepts course_id and student JWT token. Creates a Paystack transaction for the course fee and returns the payment URL for the student to complete."
💡
Paystack Fees: Paystack charges 1.5% per transaction (capped at ₦2,000) for local cards. For international cards, it is 3.9% + ₦100. There are no monthly fees. You only pay when you receive money.
12
AI Assignment Marking with Claude API — The Centrepiece
~20 minutes

This is the most powerful feature of your portal. A lecturer with 300 students cannot mark 300 essays individually and return timely, personalised feedback. Claude AI can — and it does it in seconds, consistently, at any hour of the day.

🤖
How It Works: A student submits their essay through your portal. Your Render server automatically sends the essay to Claude's API with a grading rubric you've written. Claude marks it and returns a structured score and written feedback. Your server saves the result and shows the student their grade immediately. No lecturer involvement required for routine marking.

AI Marking Flow

  1. Go to console.anthropic.com. Create a free account or log in.
  2. Click API Keys in the left sidebar. Click Create Key.
  3. Give your key a name (e.g., "training-portal"). Copy the key that appears — it starts with sk-ant-. Save it immediately — Anthropic only shows it once.
  4. In Render → Environment, add: CLAUDE_API_KEY = paste your key. Click Save Changes.
  5. Ask AI to add marking to your Flask server using this prompt:
📋 AI Marking Integration Prompt
Add an assignment submission and AI marking endpoint to my Flask server. POST /api/submit-assignment — accepts: assignment_id, student_id (from JWT), and essay_text. The server should call the Anthropic Claude API (claude-sonnet-4-20250514 model) with this system prompt:

"You are a university lecturer at a Nigerian institution. Mark the following student essay strictly and fairly. Return ONLY a valid JSON object with these fields: content_score (0-10, relevance and accuracy), argument_score (0-10, quality of reasoning), language_score (0-10, grammar and clarity), total_score (sum out of 30), grade (A/B/C/D/F), and feedback (2-3 sentences of constructive written feedback addressing the student directly)."

Save the result to the database. Return the score and feedback to the student. Handle API errors gracefully.
💡
Claude API Cost: Claude Sonnet 4 costs approximately $3 per million input tokens and $15 per million output tokens.[7] A typical 500-word student essay uses about 700 tokens. For 300 students, that is roughly 210,000 tokens — about ₦250–₦400 total to mark an entire cohort's assignments. This is significantly cheaper than any human marking arrangement at scale.
Sample AI Marking Response — What Claude Returns
{
  "content_score": 8,
  "argument_score": 6,
  "language_score": 7,
  "total_score": 21,
  "grade": "B",
  "feedback": "Your essay demonstrates a strong understanding of AI
  applications in Nigerian agriculture, particularly your analysis
  of price discovery platforms. Your argument would be strengthened
  by specific examples of existing initiatives like Farmcrowdy or
  Hello Tractor. Language use is generally clear, though more
  precise technical vocabulary would elevate your discussion."
}

Expanding What You Have Built

Your portal is now fully functional. Here is what to build next, in order of impact:

📊
Adaptive Learning Paths
Based on quiz performance, AI recommends the next module best suited to each student's level. No two students follow exactly the same path.
📋
QR Code Attendance
Students scan a QR code to check in. The server records time-stamped attendance. AI flags students below 70% attendance for early intervention.
🔍
Plagiarism Detection
Before grading, AI checks each submission against previous submissions and flags similarity above 70% for lecturer review.
💰
Fee Tracking Dashboard
Live dashboard showing paid vs outstanding students. Automated WhatsApp or SMS reminders sent to those with unpaid fees.
📝
WAEC/JAMB Practice Engine
AI generates unlimited practice questions in any subject. Marks responses instantly. Provides full explanations for wrong answers. Never runs out of questions.
📈
Analytics Dashboard
Department-level charts showing cohort performance trends, course completion rates, and at-risk student groups. Export reports as PDF.
